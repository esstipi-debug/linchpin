# SEO Modules Branding Override Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a white-label partner replace the hardcoded "Kern" string in `src/seo/feeds.py` and `src/seo/llms_txt.py`'s generated artifacts with their own brand name, via the same `src.deliverable.Branding`/`DEFAULT_BRANDING` mechanism already used by `jobs/package_deliverable.py` (E6).

**Architecture:** Add an optional `branding: Branding = DEFAULT_BRANDING` keyword parameter to the four public builder functions that currently emit a hardcoded "Kern" string. Derive the two module-level constants (`FOOTER_LINE`, `FEED_GENERATOR`) from `DEFAULT_BRANDING.name` instead of a literal, so the default output stays byte-identical while a caller can now override it per-call.

**Tech Stack:** Python 3.11, pytest, `src.deliverable.Branding` (existing frozen dataclass).

## Global Constraints

- Default behavior (no `branding` passed) MUST produce byte-identical output to today — same `llms.txt` text, same feed XML, same feed JSON.
- Reuse `src.deliverable.Branding`/`DEFAULT_BRANDING` verbatim. Do not define a new branding type.
- Zero edits to the 4 existing test assertions that reference the current default string (`tests/test_seo_feeds.py:74`, `:77`, `:131`, `tests/test_seo_llms_txt.py:96`) — only new tests are added.
- `ruff check src tests examples` stays clean throughout.
- `schema_gen.py`, `pdp_writer.py`, `crawl_audit.py`/`jobs/seo_audit.py`, `jobs/seo_priority.py`, and `geo_probe.py` are OUT OF SCOPE — none of them are touched by this plan (see spec §1 "No objetivo").
- No other file in the repo imports `src.seo.feeds` or `src.seo.llms_txt` today (verified via repo-wide grep during brainstorming) — no caller-side wiring is needed.

Full design rationale: `docs/superpowers/specs/2026-07-14-seo-branding-override-design.md`.

```python
def generate_llms_txt(site: SiteInfo, pages: Sequence[LlmsPage]) -> str:
    """Render an llms.txt document: H1 site name, blockquote summary,
    optional context paragraph, then one ``## Section`` per distinct
    ``page.section`` value (first-seen order), each listing its pages as
    markdown links, followed by :data:`FOOTER_LINE`."""
    lines: list[str] = [f"# {site.name}", "", f"> {_one_line(site.summary)}"]
    if site.context and site.context.strip():
        lines += ["", site.context.strip()]

    sections: dict[str, list[LlmsPage]] = {}
    for p in pages:
        sections.setdefault(p.section, []).append(p)

    for section_name, section_pages in sections.items():
        lines += ["", f"## {section_name}"]
        for p in section_pages:
            lines.append(f"- [{p.title}]({p.url}): {p.description}")

    lines += ["", "---", FOOTER_LINE]
    return "\n".join(lines) + "\n"
```

```python
def build_llms_txt(
    site: SiteInfo, items: Sequence[CatalogItem], *, section: str = DEFAULT_SECTION
) -> LlmsTxtReport:
    """The batch entry point: catalog -> pages/excluded -> rendered text."""
    pages, excluded = catalog_to_llms_pages(items, section=section)
    text = generate_llms_txt(site, pages)
    summary = (
        f"llms.txt: {len(pages)} page(s) listed under '{site.name}', {len(excluded)} SKU(s) excluded "
        "for missing required field(s)."
    )
    return LlmsTxtReport(
        text=text, pages=tuple(pages), excluded=tuple(excluded),
        n_pages=len(pages), n_excluded=len(excluded), summary=summary,
    )
```

```python
def build_llms_txt(
    site: SiteInfo,
    items: Sequence[CatalogItem],
    *,
    section: str = DEFAULT_SECTION,
    branding: Branding = DEFAULT_BRANDING,
) -> LlmsTxtReport:
    """The batch entry point: catalog -> pages/excluded -> rendered text."""
    pages, excluded = catalog_to_llms_pages(items, section=section)
    text = generate_llms_txt(site, pages, branding=branding)
    summary = (
        f"llms.txt: {len(pages)} page(s) listed under '{site.name}', {len(excluded)} SKU(s) excluded "
        "for missing required field(s)."
    )
    return LlmsTxtReport(
```
